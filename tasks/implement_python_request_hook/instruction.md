# Implement a Python Request Hook in Speakeasy

## Background
Speakeasy allows customizing generated SDKs using hooks. In this task, you will generate a Python SDK from an OpenAPI specification and implement a custom request hook that injects a specific header into all outgoing requests.

## Requirements
- Initialize a Speakeasy project in `/home/user/project` using the provided `openapi.yaml`.
- Configure the `gen.yaml` to generate a Python SDK named `my_sdk`.
- Enable custom hooks in the SDK configuration.
- Implement a `BeforeRequestHook` in Python that adds an `X-Organization-Id` header with the value `org_12345` to every request.
- Register the hook correctly so that it is executed by the SDK.
- Run `speakeasy generate sdk` to apply the changes.

## Implementation Guide
1. Move to `/home/user/project` where `openapi.yaml` is located.
2. Create a `gen.yaml` file configuring the Python SDK generation, ensuring hooks are enabled.
3. Run `speakeasy generate sdk` to scaffold the SDK and the `src/my_sdk/hooks` directory.
4. Create or modify the hook implementation in `src/my_sdk/hooks/` to intercept the request and append the `X-Organization-Id: org_12345` header.
5. Register your hook in `src/my_sdk/hooks/registration.py` (or the equivalent registration file generated by Speakeasy).
6. Run `speakeasy generate sdk` again to ensure the SDK is fully updated and valid.

## Constraints
- Project path: `/home/user/project`
- The header name must be exactly `X-Organization-Id`.
- The header value must be exactly `org_12345`.
- The SDK language must be Python.
